## RuyiSDK 实习岗位开放
RuyiSDK 是一个面向 RISC-V 的一体化集成开发环境（工具集）产品计划，由中国科学院软件研究所主导并开源。
计划为 RISC-V 开发者打造一套完整、易用、全栈式的开发体验，覆盖编译、调试、一键式运行（模拟运行）等关键环节，并持续支持更多主流 RISC-V 开发板和操作系统环境。
核心能力包括：组件/包管理器（ruyi）、图形化 IDE（VS Code/Eclipse 插件等）、开发者社区与技术论坛等。
项目采用快速迭代，约每个月发布一个版本，持续把最新的工具链、镜像与生态资讯带给社区与开发者。

我们的目标是：
- 降低 RISC-V 的上手门槛，帮助更多开发者“从 0 到 1”在真实设备上跑起系统和应用；
- 通过工具与文档，把复杂的开发环境搭建、运行环境部署等过程变得更简单；
- 建设一个开放、友善的开发者社区，让每位参与者都能交到同行朋友、学到真本事，并在开源世界中留下自己的足迹。

- 工作地点：线上（可远程，也欢迎本地坐班；若你所在城市有线下活动，可就近参与）
- 招聘人数：2-3人

### 岗位目标
作为产品和用户之间的“桥梁型”角色，你将与研发团队紧密配合，帮助更多开发者认识、使用并爱上 RuyiSDK。你需要先深入理解 RuyiSDK 的使用场景与产品愿景，围绕“如何使用RuyiSDK在XX开发板上开发和运行某个程序”等问题，再面向社区进行传播、运营与支持。

### 工作内容
1) 技术文档与板卡使用文档输出
  - 从“真实使用者”的视角，撰写和优化面向新手的文档内容：
    - 某款 RISC-V 开发板从开箱到点亮 RuyiSDK 的完整上手指南；
    - 在特定发行版下使用 ruyi 安装工具链、构建工程、部署镜像的图文教程；
    - 常见问题排查（如设备不识别、网络问题、环境冲突等）的 FAQ 或踩坑记录。
  - 与文档团队、开发团队协作：
    - 把社区高频问题整理成“可文档化”的条目；
    - 协助优化文档结构与示例，降低新用户的学习成本。
2) 线上社区运营与内容输出
  - 参与管理 RuyiSDK 开发者社区（论坛、微信、QQ 群组等）：
    - 及时回复用户提问，引导用户到合适的资源（文档、Issue、讨论帖）；
    - 协助整理/归档有价值的讨论，形成知识库条目。
  - 协助输出公众号文章、B站视频、图文海报等内容，向更多人介绍 RISC-V 和 RuyiSDK。
3) 代表 RuyiSDK 参与开源技术会议/沙龙，以及线下活动（以所在城市为主）
  - 在开源技术会议、沙龙、RISC-V 校园行等活动中：
    - 以线上/线下方式代表 RuyiSDK 进行技术分享或闪电演讲；
    - 制作并维护演示环境与 Demo，向参会者展示 RuyiSDK 的能力与使用体验。
  - 在你所在城市或附近有线下活动机会时：
    - 协助组织或参与本地 RISC-V Meetup、黑客松、校园招新等；
    - 收集本地用户反馈，与社区保持双向信息同步。
4) RISC-V 新闻与八卦整理
  - 持续跟踪全球 RISC-V 生态：
    - 各类发行版在 RISC-V 平台的适配进展；
    - 新硬件、新板卡、新工具链与重要安全/性能补丁；
    - 国内外重要会议、峰会、培训、论文与“有趣”的八卦故事。
  - 将零散信息整理为可读性强的简报或专题，定期发布到：
    - RuyiSDK 官方论坛“RISC-V 周报/月报”类版块；
    - 官方公众号、B 站、双周进展等渠道，帮助更多人快速了解行业动态。

## VS Code 插件开发工程师

- 招聘人数：2-3人
- 岗位编号：J163（https://github.com/plctlab/weloveinterns/blob/master/open-internships.md  J163）

### 岗位职责
- 负责 RuyiSDK VS Code 插件的功能设计、开发迭代与日常维护，能快速读懂并接手现有项目代码。
- 负责高质量交付，包含但不限于功能代码、配套的单元/集成测试以及清晰的使用文档。
- 负责知识沉淀与技术布道，将开发产出转化为面向用户和开发者的知识资源（包括但不限于使用文档、演示视频、技术博客、内外部技术交流分享等）。
- 参与工具链产品体验优化，收集反馈并持续改进插件易用性。

### 任职要求
- 技术基础：熟悉 TypeScript，具备扎实的前端/Node.js 工程开发能力，不排斥阅读和修改既有代码。
- 工程素养：具备良好的代码洁癖，注重命名规范、目录结构合理性及 Git 提交规范，坚持 KISS (Keep It Simple, Stupid) 原则，拒绝过度设计。
- 质量意识：有自觉编写测试的习惯，能通过测试用例保障代码边界与核心逻辑的可靠性。
- 文档与分享：具备技术总结与分享的意愿，能主动把“做完的事情”讲清楚，乐于通过文字、视频或技术交流等形式赋能他人。

**加分项：**
- AI 辅助开发：日常开发中允许并支持使用 AI 编码工具（如 Codex / Claude Code / OpenCode 等）。但前提是你必须能完全看懂生成的代码逻辑，并能独立承担后续的调试、修改与排错工作。
- 插件经验：有 VS Code 插件实际开发经验，熟悉 VS Code Extension API，有从 0 到 1 编写并发布过插件项目者优先。
- 领域认知：了解 RuyiSDK 和 RISC-V 工具链生态，对底层工具链或开发者体验（DX）有热情。
