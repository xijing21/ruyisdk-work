# RuyiSDK 实习生入门指南

欢迎加入 RuyiSDK！这份指南将帮助你快速了解实习期间的工作要求，特别是如何规范地记录和提交工作产出。


## 一、核心理念：把你的工作“晒”出来

在 RuyiSDK，我们非常看重**可见交付（Visible Deliverables）**。简单来说，就是你做出来的东西要能被看到、被用到、被验证。

> **完整的工作流程**：完成任务 → 撰写文档 → 发到群里 → **把链接贴进周报/月报**

**有没有产出，以你在公开渠道留下的痕迹为准。**

但也有特殊情况：如果 mentor 安排了**不适合公开的任务**（如内部调研、未公开代码库的预研等），这类产出的衡量由 mentor 直接定义，不需要勉强“晒”出来。下文会详细说明。


## 二、仓库结构

在你的 **GitHub/Gitee** 上创建一个 **public 仓库**（名字自取，如可定义为 `ruyisdk-work`），在仓库下输出周报和月报。目录结构可参考如下模版：

```
ruyisdk-work/
├── 2026-04/
│   ├── week1.md          # 4月第一周周报
│   ├── week2.md          # 4月第二周周报
│   ├── week3.md
│   ├── week4.md
│   └── month-summary.md  # 4月底月度总结
├── 2026-05/
│   ├── week1.md
│   ├── ...
│   └── month-summary.md
...

特别说明: 重点是需要输出周报和月报，目录和文件命名可以自定义，不要求与上述模版一致（仅示意，帮助理解）。

```


## 三、周报模板（按任务划分）

每周新建一个周报文档，描述清楚做了什么，给出可见产出的链接。

### 模板示例

```markdown
# 第1周周报（2026-04-01 ~ 2026-04-07）

## 任务一：在 LicheePi 4A 上跑通 RVV 向量化示例

### 相关产出（可见链接）
- PR：修复 RVV 编译错误 → https://github.com/ruyisdk/ruyisdk-sdk/pull/123
- 文档：《LicheePi 4A RVV 使用指南》 → https://github.com/ruyisdk/docs/pull/89
- 论坛分享帖：https://ruyisdk.cn/forum/topic/123

### 本周进展
已完成代码提交和文档初稿，PR 在 review 中。

---

## 任务二：解决 duo-buildroot-sdk 的编译失败问题

### 相关产出（可见链接）
- 上报 Issue：https://github.com/ruyisdk/duo-buildroot-sdk/issues/45
- 论坛回复（临时解决方案）：https://ruyisdk.cn/forum/topic/122?reply=3

### 本周进展
问题已定位，等待 maintainer 确认。

---

## 任务三：社区互动（非运营岗位额外奖励申报）

### 相关记录
- 论坛回复1：帮助解决编译问题 → https://ruyisdk.cn/forum/topic/118?reply=5
- 论坛回复2：分享 RISC-V 向量化经验 → https://ruyisdk.cn/forum/topic/119?reply=2
- 微信群帮助截图：见附件 `wechat-help-20260405.png`

### 本周总结
完成两个开发任务，社区互动累计有效回复3次。

---
备注：不适用公开的非公开性任务，具体与 mentor 协商沟通确定是否另行总结，不用在public仓库记录。

```

> 💡 **说明**：
> - **可见产出**：附上真实可点击的 URL（PR/Issue/论坛帖子等）。不能只写“修复了一个bug”而不给链接。
> - **不公开任务**：如果 mentor 明确说明“不用公开”，则不需要强行制造可见产出。此类任务的完成情况由 mentor 直接认定，且沟通和汇报方式也直接与 mentor 商议，不需要公开。

## 五、什么是“可见产出”？—— 真实具体的示例

| 类型 | 示意 |
|------|---------------------------|
| **代码 PR** | `https://github.com/ruyisdk/ruyisdk-sdk/pull/123` |
| **文档 PR** | `https://github.com/ruyisdk/docs/pull/89` |
| **Issue 反馈** | `https://github.com/ruyisdk/duo-buildroot-sdk/issues/45` |
| **论坛技术分享贴** | `https://ruyisdk.cn/forum/topic/123` |
| **论坛资讯贴** | `https://ruyisdk.cn/forum/topic/124` |
| **社区互动（论坛回复）** | `https://ruyisdk.cn/forum/topic/122?reply=3` |
| **自制教程视频** | `https://www.bilibili.com/video/BV1xx411c7mY` |
| **微信群/钉钉群活跃** | 截图并附说明，如“本周在群内帮助3位用户解决编译问题，见附件截图” |

**正确做法**：在周报/月报中，对于可见产出必须附上真实 URL；对于 mentor 安排的不公开任务，则不需要公开。


## 六、关于“社区互动奖励”

为了鼓励非运营岗位的同学参与社区互动（论坛、微信群、GitHub Issue 回复等），我们设立了额外奖励：

- **适用对象**：**非运营岗位的实习生**（开发岗、测试岗、文档岗等正式任务之外的社区贡献）。
- **奖励条件**：当月主动在论坛/微信群/Issue 下有效回复、帮助他人解决问题、发布有价值的技术贴或资讯贴，经 mentor 确认后，可获得 **最高 200 元/月** 的额外报酬（奖励有限，聊表心意和鼓励）。
- **如何申报**：在 **月报的“社区互动”任务** 一节中，进行简要说明和申请（您的奉献我们看得见）。


## 七、报表明细要求（简版）

- **每周**：新建周报，按任务划分，每个任务列出当周相关可见产出的完整 URL。原则上每周一前完成上周产出总结和更新。
- **每月**：新建月报，按任务汇总本月所有 URL，如需申报社区互动奖励也在此列出。每月最后一天更新。
- **推送提醒**：每次更新后，在实习生群里发一句“周报/月报已更新：链接”，让 mentor 知道。


## 八、几点额外提醒

**多写文档**：写完代码顺手写个使用说明，测完功能顺手写份操作教程。这些文档 PR 就是最好的可见产出。

**善用 AI**：AI 工具可以提高效率，但生成的内容要自己理解和验证，不要无脑复制。

**Git 规范**：提交 PR 时记得使用 DCO 签名（`git commit -s`）。

**论坛互动是加分项**：哪怕你不是运营，积极回复别人的问题，既锻炼自己，又能拿额外奖励。

**不公开任务怎么办**：按 mentor 要求做即可。这类任务没有公开链接是正常的，直接跟 mentor 在实习生群中沟通和汇报，由 mentor 直接评估。


## 九、常见问题

**Q：如果我这周只有调研学习，没有 PR 和 Issue，怎么办？**
A：把调研笔记、学习心得整理成一篇**论坛帖子**或 **GitHub Gist**，链接贴进周报。只要是你原创、对外可见的，就算可见产出。如果 mentor 要求此调研不公开，则按不公开任务处理，写清进展即可。

**Q：我的 PR 还没合并，可以写进周报吗？**
A：可以。写“PR #123（review 中）”，链接依然贴上。合并后再更新状态。

**Q：周报文件名可以不叫 week1/week2 吗？**
A：可以，只要你能清晰区分每周。比如 `2026-04-01_week1.md` 也行，方便理解即可。

**Q：月报里需要重复贴一遍所有链接吗？**
A：是的，需要。月度总结是为了方便 mentor 一次性查看本月所有产出，不要只写“详见各周报”。

**Q：内部分享（没有公开链接）算不算产出？**
A：算！算**有效产出**（但不叫**可见产出**）。如果任务是 mentor 安排的且不适合公开，则一切直接与 mentor 沟通商议。


**记住：在 RuyiSDK，你的仓库就是你的成绩单。每个公开链接都是你工作的证明；对于不公开的任务，诚实记录就是最好的交付。**

欢迎正式加入，期待你的精彩产出！🎉
