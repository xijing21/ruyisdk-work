# RuyiSDK 实习生培养
> 文档读者：RuyiSDK mentor

## 招募过程中的要求
实习生的招聘流程大致如下：

投递简历 → 简历筛选 → mentor 面试（含远程上机编程测试）→ 技术报告面试 → 结果通知 → 签订实习合同

目前为了 RuyiSDK 生态建设目标：
- 提高 RuyiSDK(ruyi) 下载量和装机台数 （开启遥测上传）
- 提高 RuyiSDK IDE 插件（含 VS Code 和 Eclipse ）的 install 数量
- 提升 RuyiSDK 技术论坛（ruyisdk.cn）的注册用户数

现提议：**RuyiSDK mentor 在面试的时候，建议通过给实习生布置 PreTask 任务来考察实习生的水平。并将上述的目标 “设置/嵌入” 到 PreTask 中。**

具体组织形式：在mentor负责的仓库，公开创建issue：PreTask-编号（可选）-任务名

如需要，可准备并完善 [PreTask 任务示例/任务池](https://github.com/xijing21/ruyisdk-work/blob/main/interns/pretask.md)




## 新入职统一要求
### 注重输出（文档+报告）
**要求并鼓励所有RuyiSDK项目组成员（包含员工和实习生）工作方式都把文档当成“必备项”；**
**并注重输出的传播，将“输出的文档和视频”等根据目的、受众发到不同的推广/传播渠道（官网、论坛、社群、公众号等等）**

**一件事情完成的完整流程是：完成任务 → 撰写文档 → 发到群里**

如开发岗位，写代码不是终点，写完代码后需要撰写使用说明教会项目组内部人基础的使用方法（甚至配合所负责模块的宣发和推广，如宣发文稿技术部分等）；
测试岗位，注重测试文档输出，不仅是测试的证明，也同样是新手的操作教程参考。
宣发岗位更需要注重输出（不限于文档等输出形式），结合具体的开发板、示例程序、不同的linux发行版或者运行时环境，输出具体场景下的使用案例。给新人打个样。并注重在传播渠道发送/转发。


### 注重工作汇报
面向内部（内部项目层面）和外部（开源社区领域），RuyiSDK 都有很多各种“汇报工作或者说信息同步工作”；如每周进展（内部）、RuyiSDK双周进展（公开以RuyiSDK项目角度对外）、RuyiSDK项目双周汇报（项目层面）、
每月底的实习生产出总结、各种不定时的不同汇报角度的信息同步/汇报等。这些都需要平时养成记录工作任务的习惯。

要求：
- 每个实习生在自己的github/gitee等平台创建一个 **public** 仓库（名字自取没有限制），并按周或者按月总结**可见产出/可见交付成果**。
- 每周提交《周可见产出》，每月提交《月可见产出》；（具体组织方式不限，时间节点和内容建议对应要求来；比如按月创建文档+按周更新文档内容；或者 按周创建文档+月底再新建月度总结；都行）
- 报告内容包括：
  - 每周具体工作任务简述，以及对应的可见产出成果链接。如：在XXX开发板上完成XX示例代码的构建运行，并输出文档《文章名》（链接：<附上具体链接>）
  - 每月总结：建议分类进行产出汇总，同样要求简介任务并附上可见产出链接。

> 如实习生A ,创建一个 public 仓库 ruyisdkwork，并在该仓库下按照日期创建md文档，如 2026-04.md，并按照如下内容，每周日更新一次。
```
## 2026年4月
### 第一周（起止日期）
> 起止如期不需要严格按照日历从1号开始。从上个月产出酬劳结算日算起；（基于每个产出只计算一次劳务费，可灵活设置上一个月和下一个月的分界日期；每个实习生也可不同）
### 第二周（起止日期）
### 第三周（起止日期）
...
### 月度总结

```

> 习惯把仓库当成待办+已办事项的记事本，保持记录的习惯。这也将让你受益良多。

#### 补充说明：可见产出的常见形式
- pull request （常使用简称 PR 或者 pr 代替）
- issue （针对实习生，issue 主要是上报发现的bug，或者需求建议）
- 文档仓库提交文档（PR形式；包括 docs 和 board-docs ）
- ruyisdk.cn 发帖子（技术分享贴、转发行业资讯和新闻等）
- 技术分享：在内部组织的技术分享会议上做技术分享（会酌情上传）；或者自己制作RuyiSDK使用或者RISC-V知识传播视频并推广；
- 互动和回复等活跃社区行为：包括论坛点赞回复，微信QQ群分享、回复等活跃行为。表现突出者有报酬（+200封顶）。
- 其它（要求外部可见，且有利于RuyiSDK推广和社区建设）。


## Mentor培养
不同岗位实习生技能要求不同，培养方式不同，需要mentor自行准备一份符合实习生的培养方案

### 全体都有
- 拉人入群，或创建群
- 如何快速入门 RuyiSDK ，理解 RuyiSDK 项目和我们在做什么，要做什么。
- 文档和传播意识的灌输
- RuyiSDK 生态建设思路
- git 使用技能与提升；强调 DCO 签名，git commit -s 使用；

### 文档/运营岗：J162
- Linux、RuyiSDK、编译器模拟器等技能学习和提升

### 技术岗：J159 & J163
- 了解和熟悉编程规范和要求，如何进行开发协作；如何搭建开发环境（是否需要统一）；
- 强调注重代码规范，测试和文档；
- 强调AI使用和注意事项；
