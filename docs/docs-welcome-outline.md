# 🎉 欢迎使用 RuyiSDK

RuyiSDK 是由中国科学院软件研究所主导的面向 RISC-V 开发者打造的一站式工具集和开发解决方案，让您跳过繁琐的环境搭建，直接专注于开发本身。

**RuyiSDK 正在让 RISC-V 开发变得简单**

- **一键安装**：告别繁琐配置，几分钟内完成开发环境部署
- **全面支持**：提供多样化的镜像文件、工具链、模拟器和预配置环境
- **省时高效**：将原本需要数小时的环境搭建工作简化为几个简单步骤

### 2️⃣ [快速入门](./start)

欢迎开始您的 RuyiSDK 之旅！

为照顾不同用户的需求，我们将入门流程分为两个阶段：**基础开发体验** 和 **硬件实战进阶**。您可以根据自身条件选择合适的学习路径。

:::tip 学习建议

1. 建议所有用户先完成**基础开发体验**，建立对 RISC-V 开发的基本认知。
2. 若有硬件设备，可继续进入**硬件实战进阶**，加深对硬件开发的理解。
3. 遇到问题欢迎通过[社区](https://ruyisdk.cn/c/ruyisdk/)渠道反馈，我们乐于提供帮助。

:::

**第一阶段：基础开发体验** ： PC端开发 + PC端模拟运行

> 本阶段专为没有 RISC-V 硬件设备的用户设计，通过模拟环境即可完整体验从开发到运行的全流程。

- 学习目标：
  - 掌握 RuyiSDK 包管理器安装和基础使用
  - 掌握交叉编译环境搭建
  - 理解 RISC-V 程序编译流程
  - 体验 QEMU 模拟运行环境
- [环境搭建与安装](./start/installation)
  - 操作平台：Linux x86_64（主流开发环境）
  - 安装内容：RuyiSDK 包管理器、交叉编译工具链、QEMU模拟器
- [首个项目实践](./start/first-project)
  - 实践内容：Hello World 程序开发、编译及模拟运行

**第二阶段：硬件实战进阶** ：PC端开发 + RISC-V硬件端运行

> 如果你拥有 RISC-V 硬件设备，欢迎进入本阶段，将所学应用到真实硬件上，进一步巩固开发技能。

- 学习目标：
  - 使用 RuyiSDK 为设备安装操作系统
  - 掌握硬件连接/传输/调试技巧
  - 体验真实 RISC-V 设备的运行效果
  - 感受硬件与模拟环境的差异
- [硬件设备准备](./start/hardware-setup)
  - 示例设备：以主流且便宜的 Milk-V Duo RISC-V 开发板为例
  - 配置指南：设备连接、环境配置、调试工具设置
- [硬件平台项目实践](./start/hardware-project)
  - 实践内容：将第一阶段的项目部署到真实硬件

## 📚 RuyiSDK 组件与使用说明

### RuyiSDK组成部分介绍
> - 3个基础工具；一堆被ruyi包管理器管理的基础组件；组件类型、版本、维护更新频率和计划；
3个工具组件的介绍，以及如何选择？参考 https://ruyisdk.cn/t/topic/2479 和 https://ruyisdk.cn/t/topic/2552 ；介绍不同的工具和使用方式的区别；

### RuyiSDK 软件源资源介绍
> - 基于packages-index 仓库登记的资源进行展示和介绍；让用户在使用之前就可以查询和了解资源
> - 注意：欢迎向仓库贡献资源，贡献资源的几种方式、方法；流程、注意事项。提供资源接入链接地址。
- 如何查看资源？
  - 安装之前：在官网资源页面查看软件包资源；或者直接查看packages-index仓库源文件；
  - 安装 ruyi 后：通过 ruyi list查看，各种 ruyi list的使用和组合使用；
- 资源维护策略/计划：
  - 资源分类、分级；不同类型和等级的资源维护策略；如新增资源类型和资源（用户可反馈需求提升资源维护等级、提出新资源维护需求等）、更新（更新频率，可以按照等级区别对待）、停止维护（停维策略）
  - 核心的具体的资源的维护计划和更新频率：
    - upstream: 至少每半年更新一次（可以反馈提需求）
    - ruyi-gnu/ruyi-llvm : 按照特性支持和更新情况发版；
      - ① 部分新特性支持比上游早半年左右；适合尝鲜
      - ② 愿意接收上游不愿意合入/短期不方便合入的特性或者定制扩展指令集。
    - 定制的工具链版本：如厂商扩展实现或者优化
  - 如何选择合适的版本？没有特定需求建议选择upstream，对性能等有要求，可以考虑优化版本（怎么获取哪个版本是优化版本的信息呢？教用户自己去查看和选择，或者直接给出答案）
