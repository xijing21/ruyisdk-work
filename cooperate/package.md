# RuyiSDK 接入方式

RuyiSDK 团队欢迎更多的伙伴接入 RuyiSDK 的生态，共同丰富开发资源。我们把目前支持的合作模式列举如下；我们欢迎感兴趣的合作方与我们联系。

## RuyiSDK 包管理器接入方式
RuyiSDK 包管理器通过一套工具，一个软件包索引仓库，一个RuyiSDK镜像源搭建起核心的RISC-V开发资源仓库，为RISC-V开发者提供丰富的开发资源。
工具目前主要支持Linux平台（兼容debian、ubuntu、openEuler、Fedora、Arch Linux 、Gentoo等多个主流发行版），支持amd64、arm64、riscv64三种架构，
软件包索引仓库 是“登记”软件包的元数据仓库，是一个软件包的描述信息。支持“RuyiSDK镜像源”和其它镜像源/分发平台。
RuyiSDK镜像源：托管和分发开发资源文件，提供资源的托管和下载服务。支持源码、二进制、镜像、示例代码等多种资源类型和压缩格式；

RuyiSDK 包管理器对资源的集成接入很灵活，主要分为以下两种方式：

### 源码对接

* 您（作为资源提供方，下同）需要提供软件包的**源码** （适用于源码开放的情况）。
* **RuyiSDK 团队**将其构建为二进制包。
* 二进制包通过 **RuyiSDK 官方镜像源** 分发（这种分发渠道定义通过 **RuyiSDK 官方软件源仓库**分发）。
* 授权：您需要授权（需要什么授权？）

这种方式适合没有分发平台，且没有对源码进行构建的情况。资源提供方可向 RuyiSDK 团队分享源码以及打包的必要做法与建议等信息，以便 RuyiSDK 团队的打包版本能更好体现原产品的风貌。

### 二进制包对接

在源码对接模式不适用的场合，RuyiSDK 也可以单纯做二进制包的分发。您需要把相关软件包的二进制兼容情况告知
RuyiSDK 团队，如支持架构、支持发行版等，以便 RuyiSDK 团队进行测试等工作。

此种对接方式下，关于二进制包的分发方式，目前存在两种选择：

* 您提供相关软件包的**二进制分发版本**，并授权 RuyiSDK 团队对其进行再分发。
* **RuyiSDK 团队**负责将其上传至 RuyiSDK 官方镜像源。
* 二进制包通过 **RuyiSDK 官方镜像源**被再分发。
* 软件包的定义通过 **RuyiSDK 官方软件源仓库**分发。

以及：

* 您自行维护相关软件包的**下载链接**，并提供给 RuyiSDK 团队。
* 二进制包通过其 **自有渠道**分发。
* 软件包的定义通过 **RuyiSDK 官方软件源仓库**分发。

如您欲自行维护相关资源下载，需要注意放行 RuyiSDK 包管理器的下载请求：目前 RuyiSDK 包管理器不支持特殊的鉴权或登录流程，在服务器看来就是一般的
`wget` 或 `curl` 之类的请求，但会携带一个 `User-Agent: ruyi/<包管理器版本号>` 的 HTTP 头。
