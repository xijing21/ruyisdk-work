# RuyiSDK 用户注册管理

## 背景
为了满足RuyiSDK社区推广和用户统计的需求，针对RuyiSDK产品（包括包管理器、IDE、官网、技术论坛、Github代码仓库、微信公众号RUYISDK）进行用户注册/登录管理，以 **引导** 为主，而非 **强制** ，引导用户登录，积累RuyiSDK用户量，作为后续RuyiSDK社区规模衡量参考之一。

**用户规模相关统计现状**：

1. 基于ruyisdk.cn 技术论坛（自助搭建的Discourse论坛），建立一个基于邮箱+密码注册的简易用户信息。
2. 基于 ruyi 包管理遥测功能收集的包管理器安装量、下载量（如下所示），参考：https://ruyisdk.org/dashboard

3. 官网 ruyisdk.org ：暂未集成用户注册功能

5. RUYISDK微信公众号：未提供注册服务；其它版块也未有与微信授权相关的联动

## RuyiSDK 引导注册方案

- 在技术论坛Discourse上启用并配置OAuth服务端功能，使Discourse能够作为一个OAuth 2.0提供商，接受来自IDE、官网等客户端的授权请求。在IDE插件和官网中加入引导用户注册的入口。
- github 作为 ruyisdk 的代码托管平台，目前托管了 40+ 个 ruyisdk 的仓库；后续可以考虑通过 github 授权的方式登录ruyisdk。统一配置在Discourse后台，由Discourse作为“网关”接入。
- 微信授权：RUYISDK 公众号目前也积累了一些用户，作为目前的主流讯息入口之一，后续可以考虑通过 微信授权方式登录ruyisdk，在中国地区这种授权登录方式将很有优势。（芯来科技采用的这一方式）
- 用户数据：只存在Discourse的数据库里。且只采集最小用户数据：邮箱和密码（或者github/微信授权信息）

这样做的好处是：
- 不改现有论坛：论坛老用户不受影响，直接用原有账号密码登录所有新地方。
- 一次接入，全网通用：只需开发一次Discourse的OAuth客户端，所有产品都复用。
- 降低合规风险：你依然只维护Discourse那一套用户数据，没有新增数据存储点。
